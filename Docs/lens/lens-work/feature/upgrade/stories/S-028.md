## Description

Implement structured error messages for all gate enforcement failures following the UX Design §2 gate feedback contract. Every error includes: the specific check that failed, why it matters, and what action resolves it.

## Acceptance Criteria

- [ ] Constitution violation: shows specific rule and permitted values
- [ ] State drift: shows divergent fields, suggests /sync
- [ ] All errors are structured (not prose), actionable, and specific
