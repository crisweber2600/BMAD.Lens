# S-027: Implement /status command (3-line + verbose)

**Epic:** E-007
**Priority:** P1
**Story Points:** 3
**Dependencies:** S-003, S-004, S-018

## Description

Implement the `/status` command per NFR10 specification: 3-line structured summary showing initiative, current state, and next action. Verbose mode (`/status -v`) shows all phases, gates, branches, and event count.

## Acceptance Criteria

- [ ] Default output: 3-line format per NFR10
- [ ] Line 1: `{id} ({track}) — {phase} [{status}] on {audience} ({label})`
- [ ] Line 2: Pending item or 'All clear'
- [ ] Line 3: `Next: {specific_next_action}`
- [ ] Verbose mode: all phases with status, all gates, branch names, event count
- [ ] Shows all initiatives if called without active initiative
- [ ] Detects and surfaces state drift (suggests /sync)

- Implementation target: `_bmad/_config/custom/lens-work/`
- Test approach: Manual validation + dogfood usage
