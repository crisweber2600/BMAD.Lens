# S-011: Implement branch naming validation

**Epic:** E-003
**Priority:** P1
**Story Points:** 3
**Dependencies:** S-008

## Description

Implement validation of branch names against naming conventions and platform limits. All branch names must use flat hyphen-separated format (no `/` separators) and must not exceed platform MAX_PATH limits (especially relevant on Windows).

## Acceptance Criteria

- [ ] Validates branch names use only lowercase alphanumeric + hyphens
- [ ] Rejects branch names containing `/` (non-flat)
- [ ] Warns when branch name approaches MAX_PATH limits (Windows: 260 chars)
- [ ] Validates initiative ID suffix is exactly 6 characters
- [ ] Validation runs before any branch creation

- Test approach: Manual validation + dogfood usage
