# S-025: Implement precondition validation chain

**Epic:** E-007
**Priority:** P1
**Story Points:** 5
**Dependencies:** S-003, S-004, S-015

## Description

Implement the precondition validation that runs before every phase dispatch: (1) previous phase must be complete (FR3), (2) required artifacts must exist, (3) constitution must permit the action. This is the core enforcement mechanism for FT1.

## Acceptance Criteria

- [ ] Checks previous phase status is 'complete' before dispatching next phase
- [ ] Checks required artifacts from prior phases exist
- [ ] Checks constitution permits the track/action
- [ ] Returns structured error per UX gate feedback contract on any failure
- [ ] Validation chain runs for all phase commands, not just /next
- [ ] Handles first phase (preplan) which has no previous phase
