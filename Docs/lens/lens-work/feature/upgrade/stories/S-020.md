# S-020: Implement /onboard flow (4-step)

**Epic:** E-005
**Priority:** P2

## Description

Implement the `/onboard` command following the 4-step flow defined in UX Design §1: (1) detect existing initiatives, (2) teach three concepts (phases, tracks, audiences), (3) show Tier 1 commands, (4) offer to create or switch.

## Acceptance Criteria

- [ ] Step 1: Detects returning vs. new user (initiatives exist or not)
- [ ] Step 2: Explains phases, tracks, audiences in 3 concise paragraphs
- [ ] Step 3: Shows Tier 1 commands: /new, /status, /switch
- [ ] Step 4: Offers /new (new user) or /switch (returning user)
- [ ] Completes in ≤5 user interactions (NFR12)
- [ ] Re-runnable at any time
