# S-015: Implement gate validation for tracks, artifacts, and reviewers

**Epic:** E-004
**Priority:** P1
**Story Points:** 3
**Dependencies:** S-014

## Description

Implement gate validation that checks the resolved constitution at transition points: (1) validate track is permitted, (2) validate required artifacts exist, (3) validate required reviewers are present for audience promotions.

## Acceptance Criteria

- [ ] Track validation: initiative's track must be in resolved `permitted_tracks` list
- [ ] Artifact validation: all `required_artifacts[phase]` must exist at expected paths
- [ ] Reviewer validation: PR reviewers must include all `required_reviewers[promotion]`
- [ ] Each validation returns pass/block with specific failure reason
- [ ] Validation result logged in event log (FR24)
- [ ] Default: no constitution at any level → all gates pass (lifecycle.yaml defaults apply)
