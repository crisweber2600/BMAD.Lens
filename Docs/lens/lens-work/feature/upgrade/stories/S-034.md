# S-034: Implement /new (initiative creation)

**Epic:** E-008
**Priority:** P1
**Story Points:** 4
**Dependencies:** S-003, S-004, S-006, S-008

## Description

Implement the `/new` command that creates a complete initiative: prompts for domain/service/feature/track, creates initiative config from template, creates audience branches (small/medium/large), updates state, logs event. Includes confirmation prompt per UX Design.

## Acceptance Criteria

- [ ] Prompts for: domain, service, feature name, track selection
- [ ] Generates initiative ID: `{feature-slug}-{6char_random}`
- [ ] Creates initiative config from template with all v2 fields
- [ ] Creates audience branches (small/medium/large) from initiative root
- [ ] Creates initiative root branch from current branch
- [ ] Updates state.yaml: active_initiative, current_phase (first phase of track)
- [ ] Confirmation prompt: 'Create initiative X with track Y? [y/n]'
- [ ] Logs init_initiative event
