# S-009: Implement phase branch lifecycle

**Epic:** E-003
**Priority:** P1
**Story Points:** 5
**Dependencies:** S-008

## Description

Implement the full lifecycle of phase branches: creation from audience branch when a phase starts, checkout, and optional deletion after PR merge. Phase branches follow the pattern `{root}-{audience}-{phase}`.

## Acceptance Criteria

- [ ] Can create phase branch from audience branch: `{root}-{audience}-{phase}`
- [ ] Can checkout existing phase branch and pull latest
- [ ] Can push phase branch to remote
- [ ] Can detect if phase branch already exists (idempotent creation)
- [ ] Can delete phase branch after PR merge (optional cleanup via FR10)
- [ ] Phase branch created from correct audience (derives from lifecycle.yaml)
