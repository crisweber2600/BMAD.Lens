# S-013: Implement constitution file loading from 4-level hierarchy

**Epic:** E-004
**Priority:** P1
**Story Points:** 3
**Dependencies:** None

## Description

Implement loading of constitution files from the four-level LENS hierarchy: org → domain → service → repo. Constitution files are Markdown with embedded YAML at `_bmad-output/lens-work/constitutions/{level}/{path}/constitution.md`.

## Acceptance Criteria

- [ ] Can load constitution from org level (if exists)
- [ ] Can load constitution from domain level: `constitutions/domain/{domain}/constitution.md`
- [ ] Can load constitution from service level: `constitutions/service/{domain}/{service}/constitution.md`
- [ ] Can load constitution from repo level: `constitutions/repo/{domain}/{service}/{repo}/constitution.md`
- [ ] Missing levels contribute empty set (no error)
- [ ] Can parse YAML code blocks within Markdown constitution files

## Technical Notes

- Implementation target: `_bmad/_config/custom/lens-work/`
- Test approach: Manual validation + dogfood usage
- Architecture reference: architecture.md
