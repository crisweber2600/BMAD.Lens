# S-036: Implement /fix (3-category repair)

**Epic:** E-009
**Priority:** P2
**Story Points:** 3
**Dependencies:** S-035

## Description

Implement the `/fix` command that detects and repairs three categories of state issues: (1) state drift — PR merged but status not updated, (2) orphan branches — branches exist without state entries, (3) orphan state — state references non-existent branches.

## Acceptance Criteria

- [ ] Category 1: Detects PRs merged but phase_status still 'pr_pending' → updates to 'complete'
- [ ] Category 2: Detects branches with no matching state/config entry → offers deletion
- [ ] Category 3: Detects state entries referencing non-existent branches → clears stale entries
- [ ] All changes previewed before applying
- [ ] User confirms each fix category
- [ ] Logs state_fix event per category

- Implementation target: `_bmad/_config/custom/lens-work/`
- Test approach: Manual validation + dogfood usage
