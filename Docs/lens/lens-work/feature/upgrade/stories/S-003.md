# S-003: Implement state.yaml read/write with v2 schema

**Epic:** E-002
**Priority:** P1
**Story Points:** 5
**Dependencies:** None

## Description

Implement the state management skill's ability to load and save `_bmad-output/lens-work/state.yaml` using the v2 lifecycle contract schema. The v2 schema uses named phases (preplan, businessplan, techplan, devproposal, sprintplan) instead of numbered phases, includes `lifecycle_version: 2`, and tracks `audience_status` for promotion gates.

## Acceptance Criteria

- [ ] Can load state.yaml and parse all v2 fields: lifecycle_version, active_initiative, current_phase, active_track, workflow_status, phase_status map, audience_status map
- [ ] Can save state.yaml preserving all fields and comments
- [ ] Schema validation rejects v1-format state files with clear error
- [ ] Schema validation rejects invalid phase names (must be canonical set)
- [ ] Default state template matches `templates/state-template.yaml`

## Technical Notes

- Implementation target: `_bmad/_config/custom/lens-work/`
- Test approach: Manual validation + dogfood usage
- Architecture reference: architecture.md
