# S-006: Implement event log append (JSONL)

**Epic:** E-002
**Priority:** P1
**Story Points:** 3
**Dependencies:** None

## Description

Implement the append-only event log at `_bmad-output/lens-work/event-log.jsonl`. Each event is a single JSON line following the schema defined in architecture §4.3. The log must support all event types: init_initiative, phase_start, phase_pr_created, phase_complete, party_mode_review, audience_promotion, constitution_check, state_sync, state_fix, bootstrap.

## Acceptance Criteria

- [ ] Can append a single JSON line to event-log.jsonl
- [ ] Each event includes: ts (ISO8601), event (type), initiative (id)
- [ ] All 10 event types from §4.3 are supported
- [ ] Events are immutable — no retroactive edits (append-only)
- [ ] Event log is parseable line-by-line (each line is valid JSON)
- [ ] Can read and filter events for a specific initiative (FR36)

## Technical Notes

- Implementation target: `_bmad/_config/custom/lens-work/`
- Test approach: Manual validation + dogfood usage
- Architecture reference: architecture.md
