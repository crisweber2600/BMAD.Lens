# S-005: Implement dual-write contract enforcement

**Epic:** E-002
**Priority:** P1
**Story Points:** 3
**Dependencies:** S-003, S-004

## Description

Implement the dual-write contract that ensures every mutation to `current_phase`, `phase_status`, or `gate_status` updates BOTH state.yaml AND the active initiative's config file. No single-file mutations are permitted for these shared fields.

## Acceptance Criteria

- [ ] Every state mutation function updates both state.yaml and initiatives/{id}.yaml
- [ ] Shared fields (current_phase, phase_status) are identical in both files after any mutation
- [ ] Mutation fails atomically if either file write fails (compensating action)
- [ ] Dual-write is the ONLY way to update shared fields — no direct single-file writes
- [ ] Source of truth hierarchy documented: git > event log > state files

- Test approach: Manual validation + dogfood usage
