# S-017: Implement artifact path resolution

**Epic:** E-005
**Priority:** P2
**Story Points:** 3
**Dependencies:** S-004

## Description

Implement resolution of artifact paths from the initiative config's `docs` block. All artifacts must resolve to `Docs/{domain}/{service}/feature/{feature}/` pattern. Cross-phase artifact loading uses these resolved paths.

## Acceptance Criteria

- [ ] Can resolve artifact path: `{docs_path}/{artifact-name}.md`
- [ ] Path uses normalized pattern: `Docs/{domain}/{service}/feature/{feature}/`
- [ ] Can verify artifact exists at resolved path (file existence check)
- [ ] Can load artifact content from resolved path
- [ ] Handles both file artifacts (.md) and directory artifacts (stories/)
