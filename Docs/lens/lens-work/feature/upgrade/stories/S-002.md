# S-002: Add PR-gate enforcement to all phase exits

**Epic:** E-001
**Priority:** P0
**Story Points:** 2
**Dependencies:** S-001

## Description

After auto-advance patterns are removed (S-001), add explicit PR-gate enforcement to every phase exit point. Each phase workflow must end by creating a PR and setting status to `pr_pending` — phase cannot be marked `complete` until the PR is merged. This implements passive phase-completion detection per architecture §8.4.

## Acceptance Criteria

- [ ] Every phase workflow ends with PR creation step (phase branch → audience branch)
- [ ] Phase status transitions: `in_progress` → `pr_pending` (on PR create) → `complete` (on merge detect)
- [ ] No workflow sets phase status to `complete` directly — only merge detection does
- [ ] Merge detection uses `git merge-base --is-ancestor` per §8.4
