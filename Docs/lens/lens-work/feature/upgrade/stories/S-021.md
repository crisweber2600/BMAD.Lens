# S-021: Implement checklist schema and gate logic

**Epic:** E-006
**Priority:** P2
**Story Points:** 3
**Dependencies:** S-015

## Description

Implement the progressive checklist schema from architecture §3.2.6. The checklist tracks gate readiness items (passed/pending/failed) and calculates gate_ready percentage. Gate is ready when all required items are passed.

## Acceptance Criteria

- [ ] Checklist schema matches §3.2.6: items array with status, required flag, detected_at
- [ ] Gate readiness = (passed required items / total required items) * 100
- [ ] Gate blocks when any `required: true` item is not `passed`
- [ ] Non-required items are informational (don't block)
- [ ] Checklist items generated from resolved constitution's required_artifacts
