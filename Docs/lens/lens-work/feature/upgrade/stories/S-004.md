# S-004: Implement initiative config CRUD

**Epic:** E-002
**Priority:** P1
**Story Points:** 5
**Dependencies:** None

## Description

Implement create, read, update operations for per-initiative configuration files at `_bmad-output/lens-work/initiatives/{id}.yaml`. Each initiative config stores the track, phase status, docs path, branch root, and gate status for one initiative.

## Acceptance Criteria

- [ ] Can create initiative config from template with all required fields
- [ ] Can read and parse initiative config YAML
- [ ] Can update individual fields (phase_status, current_phase, gate_status)
- [ ] Config stored at `_bmad-output/lens-work/initiatives/{id}.yaml`
- [ ] Config schema matches initiative-template.yaml with all v2 fields
- [ ] Feature-level initiatives stored flat (not nested under domain/service)

## Technical Notes

- Implementation target: `_bmad/_config/custom/lens-work/`
- Test approach: Manual validation + dogfood usage
- Architecture reference: architecture.md
