# S-010: Implement PR creation with GitHub MCP and fallback

**Epic:** E-003
**Priority:** P1
**Story Points:** 5
**Dependencies:** S-009

## Description

Implement PR creation for two use cases: (1) phase branch → audience branch (phase completion), and (2) audience branch → next audience branch (audience promotion). Use GitHub MCP tools when available; fall back to manual instructions when API is unavailable.

## Acceptance Criteria

- [ ] Can create PR: phase branch → audience branch with structured title and body
- [ ] Can create PR: audience branch → audience branch for promotions
- [ ] PR title follows convention: `phase({name}): Complete {phase} for {initiative}`
- [ ] PR body includes initiative context, artifacts produced, next steps
- [ ] Graceful fallback when GitHub MCP unavailable: prints manual PR URL and instructions
- [ ] PR URL captured and stored in state (phase_status.pr_url)
