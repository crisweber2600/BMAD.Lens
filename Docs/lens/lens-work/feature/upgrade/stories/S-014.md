# S-014: Implement additive merge algorithm (set-union)

**Epic:** E-004
**Priority:** P1
**Story Points:** 5
**Dependencies:** S-013

## Description

Implement the additive merge algorithm that combines constitutions from all LENS levels using set-union semantics. Lower levels can only ADD requirements, never remove them. The merge must be deterministic — same inputs always produce same output.

## Acceptance Criteria

- [ ] List-valued fields merged using deduplicated set-union
- [ ] Dict-valued fields merged recursively with same semantics
- [ ] Scalar fields: most specific level wins (repo > service > domain > org)
- [ ] Lower level cannot remove items added by higher level
- [ ] Merge is deterministic and idempotent
- [ ] Example: domain sets [pm, architect], service adds [pm, qa] → resolved: [pm, architect, qa]
